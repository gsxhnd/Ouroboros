# 10 - 设计决策与注意事项

## 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 应用 crate 结构 | 单一 `ourboros` binary + 子命令 | `server` / `app` 共用 HTTP 与 API 代码，减少重复启动逻辑 |
| core 与 HTTP 分层 | `ourboros-core` 独立 lib | 领域逻辑与传输层分离，便于测试与未来 CLI / worker 复用 |
| 桌面端架构 | 系统托盘 + 内嵌 HTTP Server | 纯 Rust 实现，无 WebView 依赖，用户通过浏览器访问界面 |
| 系统托盘库 | tray-icon + muda + tao | 跨平台支持好，Tauri 团队维护，轻量活跃；**不使用 Tauri 框架** |
| Web UI 打包 | dist 外置磁盘托管，不 embed | 便于用户替换第三方 UI；改前端无需重编 Rust |
| 托盘依赖隔离 | Cargo feature `tray` | NAS 构建 `--no-default-features` 避免 GTK/tao 等桌面依赖 |
| ORM | SeaORM | 异步原生、多数据库后端（SQLite/PG/MySQL）、类型安全 |
| 数据库迁移 | sea-orm-migration | 与 SeaORM 深度集成，代码化迁移，多后端兼容 |
| 数据库 | SQLite 为主，PG/MySQL 可选 | 嵌入式无需部署，服务化时通过 database_url 切换 |
| API 协议 | REST (Axum) + WebSocket | REST 处理 CRUD 请求，WebSocket 推送实时事件；不使用 gRPC |
| 前端数据请求 | zustand + fetch | 由 store 直接管理请求状态，不引入 react-query |
| 文件策略 | 不转移文件 | 尊重用户文件组织习惯，减少数据风险 |
| 缩略图存储 | 资源库 .ourboros/thumbnails/ | 随资源库一起迁移、备份 |
| 前端状态 | zustand | 轻量、TypeScript 友好、无 boilerplate |
| 前端 UI | shadcn/ui | 可定制、无运行时依赖、Tailwind 集成 |
| 资源库模型 | 一实例一库，可切换 | 简化并发问题，每个 library 独立数据库 |
| core 接口 | 全异步 (async fn) | SeaORM 原生异步，配合 tokio 生态 |
| 并发控制 | tokio::sync::RwLock | 替代 std::sync::Mutex，支持异步读写分离 |

## 平台适配决策

| 平台 | 方案 | 效果 |
|------|------|------|
| macOS | .app bundle + 启动脚本 `ourboros app` + LSUIElement + ActivationPolicy::Accessory | 不弹终端、不显示 Dock 图标 |
| Windows | 运行 `ourboros app` | 托盘驻留，浏览器访问 UI |
| Linux | .desktop 启动 `ourboros app`，依赖 GTK3 | 正常显示托盘图标 |

## 注意事项

### 性能

1. **文件监听**：大型资源库（数万文件）需注意 watcher 的内存占用，考虑分级监听或延迟索引
2. **缩略图生成**：使用后台线程池异步生成，不阻塞主线程和 API 响应
3. **虚拟滚动**：前端必须使用虚拟列表，避免一次性渲染大量 DOM
4. **RAW 格式**：RAW 解码计算量大，考虑缓存解码结果

### 数据安全

5. **数据库迁移**：使用 sea-orm-migration 管理 schema 变更，确保向前兼容
6. **多实例安全**：SQLite WAL 模式处理并发，需确保同一资源库不被多个实例同时写入
7. **文件删除**：默认仅从数据库移除索引，需用户二次确认才删除实际文件

### 跨平台

8. **路径处理**：统一使用 `/` 分隔符存储，运行时按系统转换
9. **文件系统差异**：注意 macOS (APFS) / Windows (NTFS) / Linux (ext4) 对大小写敏感性差异
10. **Linux 托盘**：依赖 `libgtk-3-dev`（编译时）和 GTK3 运行时

### 架构

11. **tao 事件循环 + tokio**：`app` 模式下 tao 主线程阻塞运行事件循环，tokio runtime 在独立线程中 spawn，通过 oneshot channel 优雅关闭
12. **错误处理**：core 层使用 `thiserror` 定义领域错误，HTTP 层转换为 HTTP 状态码
13. **配置管理**：支持环境变量 + 配置文件双重来源；Web UI 目录支持 `OURBOROS_WEB_DIR` 环境变量
14. **FTS5 兼容性**：SQLite FTS5 需要 raw SQL 处理，PostgreSQL 场景使用 tsvector/tsquery
15. **静态 UI 路径**：发布包需确保 `web/dist`（或自定义 UI）与 binary 相对路径正确；macOS 打包脚本复制至 `Contents/Resources/web/`
