# 07 - 数据库设计

## 存储方案

- 使用 **SeaORM** 作为 ORM 框架，支持多数据库后端
- 默认使用 SQLite（嵌入式，无需额外部署）
- 服务化部署时可切换到 PostgreSQL 或 MySQL（通过配置 `database_url`）
- 使用 `sea-orm-migration` 管理 schema 变更
- 每个资源库独立一个数据库文件（SQLite 场景下为 `<library>/.ourboros/database.db`）

## ORM 架构

```
ourboros-entity/     → SeaORM Entity 定义（表结构映射）
ourboros-migration/  → 数据库迁移（sea-orm-migration）
ourboros-core/       → 业务逻辑层，通过 DatabaseConnection 操作数据
```

## 数据库连接

| 后端 | 连接 URL 示例 |
|------|---------------|
| SQLite | `sqlite:/path/to/.ourboros/database.db?mode=rwc` |
| PostgreSQL | `postgres://user:pass@localhost:5432/ourboros` |
| MySQL | `mysql://user:pass@localhost:3306/ourboros` |

## 核心表结构

### assets - 资源表

```sql
CREATE TABLE assets (
    id TEXT PRIMARY KEY,
    file_path TEXT NOT NULL UNIQUE,
    file_name TEXT NOT NULL,
    file_ext TEXT NOT NULL,
    file_size INTEGER NOT NULL,
    file_hash TEXT,
    mime_type TEXT NOT NULL,
    asset_type TEXT NOT NULL,
    width INTEGER,
    height INTEGER,
    duration REAL,
    rating INTEGER DEFAULT 0,
    color_hex TEXT,
    color_palette TEXT,
    notes TEXT,
    metadata_json TEXT,
    is_deleted INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    modified_at TEXT NOT NULL,
    indexed_at TEXT NOT NULL
);
```

### folders - 虚拟文件夹

```sql
CREATE TABLE folders (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    parent_id TEXT REFERENCES folders(id),
    color TEXT,
    icon TEXT,
    sort_order INTEGER DEFAULT 0,
    created_at TEXT NOT NULL
);
```

### asset_folders - 资源-文件夹关联

```sql
CREATE TABLE asset_folders (
    asset_id TEXT REFERENCES assets(id) ON DELETE CASCADE,
    folder_id TEXT REFERENCES folders(id) ON DELETE CASCADE,
    PRIMARY KEY (asset_id, folder_id)
);
```

### tags - 标签

```sql
CREATE TABLE tags (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    group_name TEXT,
    color TEXT,
    created_at TEXT NOT NULL
);
```

### asset_tags - 资源-标签关联

```sql
CREATE TABLE asset_tags (
    asset_id TEXT REFERENCES assets(id) ON DELETE CASCADE,
    tag_id TEXT REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (asset_id, tag_id)
);
```

### smart_folders - 智能文件夹

```sql
CREATE TABLE smart_folders (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    rules_json TEXT NOT NULL,
    sort_order INTEGER DEFAULT 0,
    created_at TEXT NOT NULL
);
```

### 全文搜索索引

```sql
CREATE VIRTUAL TABLE assets_fts USING fts5(
    file_name, notes, tags_text,
    content='assets',
    content_rowid='rowid'
);
```

## 索引

```sql
CREATE INDEX idx_assets_type ON assets(asset_type);
CREATE INDEX idx_assets_ext ON assets(file_ext);
CREATE INDEX idx_assets_rating ON assets(rating);
CREATE INDEX idx_assets_color ON assets(color_hex);
CREATE INDEX idx_assets_created ON assets(created_at);
CREATE INDEX idx_assets_hash ON assets(file_hash);
CREATE INDEX idx_tags_group ON tags(group_name);
```
