## 单一二进制 + 子命令

`ourboros` crate 提供**一个可执行文件**，通过 clap 子命令区分运行模式：

| 子命令 | 用途 | 托盘 | 典型场景 |
|--------|------|------|----------|
| `ourboros server` | HTTP 服务（无 GUI） | 否 | NAS、Docker、systemd |
| `ourboros app` | 系统托盘 + 内嵌 HTTP | 是 | macOS / Windows / Linux 桌面 |

```bash
# 服务端（NAS / CLI）
ourboros server --host 0.0.0.0 --port 8080 --web-dir ./web/dist

# 桌面（系统托盘）
ourboros app
ourboros app --web-dir /path/to/custom-ui
```

**Cargo feature `tray`**：托盘相关依赖（tray-icon、muda、tao 等）为 optional。NAS 构建时禁用默认 feature，避免引入 GTK 等桌面依赖：

```bash
cargo build -p ourboros --no-default-features --release   # 仅 server 子命令
cargo build -p ourboros --release                        # server + app
```

## 桌面端架构（`ourboros app`）

```
┌──────────────────────────────────────────┐
│              ourboros app                   │
│                                          │
│  tray-icon + muda (菜单) + tao (事件循环) │
│         │                                │
│         ├── 启动/停止 HTTP Server         │
│         ├── 打开浏览器访问界面            │
│         └── 退出应用                     │
│                                          │
│  内嵌 Axum HTTP server（127.0.0.1:随机端口）│
└──────────────────────────────────────────┘
```

不使用 Tauri / WebView；用户通过系统浏览器访问 Web UI。

## Web UI 托管策略

前端构建产物 `web/dist` **不 embed 进二进制**，运行时从磁盘读取，便于替换第三方 UI：

| 优先级 | 来源 |
|--------|------|
| 1 | CLI `--web-dir` |
| 2 | 环境变量 `OURBOROS_WEB_DIR` |
| 3 | 自动探测：可执行文件旁 `web/`、macOS `../Resources/web` 等 |

未指定或找不到有效目录（需含 `index.html`）时，进入 **API-only** 模式，仅提供 REST API。

Axum 路由：`/api/*`、`/health` 走 API；其余路径 fallback 到静态文件（SPA → `index.html`）。

**开发模式**仍使用 Vite dev server（`:1420`）+ proxy 到后端，无需每次 build dist。

## 部署模式

| 模式 | 命令 | 说明 |
|------|------|------|
| **桌面端 (系统托盘)** | `ourboros app` | 驻留系统托盘，内嵌 HTTP（localhost 随机端口）。托盘菜单可打开浏览器。macOS 打包为 `.app` bundle（LSUIElement）。 |
| **服务端** | `ourboros server` | 监听固定端口，适合 NAS / 团队共享。可配合 `--web-dir` 托管官方或第三方 UI。 |
| **纯 API** | `ourboros server`（无 `--web-dir`） | 仅 REST API；静态文件由 Nginx 等外部服务托管。 |

## 平台适配

| 平台 | 适配方案 |
|------|----------|
| **macOS** | `.app` bundle + 启动脚本执行 `ourboros app` + `Info.plist` 设置 `LSUIElement=true` + `ActivationPolicy::Accessory`，不显示 Dock 图标 |
| **Windows** | 直接运行 `ourboros app`，托盘图标驻留 |
| **Linux** | 通过 `.desktop` 文件启动 `ourboros app`，依赖 GTK3 显示托盘图标 |
