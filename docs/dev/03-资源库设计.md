# 03 - 资源库设计

## 资源库概念

- 一个资源库 = 一个用户指定的根目录
- 一个运行实例对应一个资源库
- 桌面端可同时打开多个实例（多窗口/多进程），每个对应不同资源库
- 支持在应用内切换资源库（关闭当前 → 打开另一个）

## 资源库目录结构

```
/用户指定的资源库目录/
├── .ourboros/                  # 资源库元数据（隐藏目录）
│   ├── database.db             # SQLite 数据库
│   ├── config.toml             # 资源库配置
│   └── thumbnails/             # 缩略图缓存
│       ├── {asset_id}_sm.webp  # 小缩略图 (200px)
│       ├── {asset_id}_md.webp  # 中缩略图 (400px)
│       └── {asset_id}_lg.webp  # 大缩略图 (800px)
├── photos/                     # 用户的实际文件（不做任何修改）
│   ├── 2024/
│   │   └── vacation.jpg
│   └── portraits/
│       └── headshot.png
├── documents/
│   └── contract.pdf
└── ...
```

## 文件路径映射

- 数据库中存储相对于资源库根目录的相对路径
- 实际路径 = 资源库根目录 + 相对路径
- 不复制、不移动用户文件
- 跨平台路径：统一使用 `/` 分隔符存储，运行时按系统转换

## 资源库配置文件 (.ourboros/config.toml)

```toml
[library]
name = "我的素材库"
version = "1.0.0"
created_at = "2024-01-01T00:00:00Z"

[watch]
enabled = true
ignored_patterns = [".DS_Store", "Thumbs.db", ".git"]
debounce_ms = 500

[thumbnail]
small = 200
medium = 400
large = 800
format = "webp"
quality = 80
```
